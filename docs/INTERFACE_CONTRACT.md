# CIC Sandbox Interface Contract

This contract defines the boundary for CIC downsampling work.

## Module under test

- Entity: `CIC_N2_GEN_TRUNC`
- File: `vhdl/CIC_N2_GEN_TRUNC.vhd`

## Generic parameters

- `in_bit` (positive): input sample width in bits.
- `out_bit` (positive): output sample width in bits.
- `out_rate` (positive): log2 decimation factor used by sizing (`in_bit + 2*out_rate` for internal integrator width in this implementation).
  In the original `rpll_fpga` design, this is intended to match the slow-strobe cadence (`Rate_CLK_D`).

## Ports

- `Reset` (`in std_logic`): asynchronous active-high reset.
- `Data` (`in std_logic_vector(in_bit-1 downto 0)`): signed two's-complement input sample.
- `Clock` (`in std_logic`): fast processing clock.
- `Clock_Slow` (`in std_logic`): slow strobe/enable pulse in `Clock` domain.
- `Clock_out` (`out std_logic`): pass-through of `Clock_Slow`.
- `Qout` (`out std_logic_vector(out_bit-1 downto 0)`): signed two's-complement decimated output.

## Clocking and timing assumptions

- `Clock_Slow` is synchronous to `Clock`.
- `Clock_Slow` is expected to be a one-cycle pulse at the desired decimation cadence.
- Integrators run every `Clock`.
- Comb/output registers update only when `Clock_Slow = '1'`.

## `Clock_Slow` meaning in original `rpll_fpga`

- `Clock_Slow` is a strobe/enable pulse, not a separate low-frequency clock domain.
- In the PLL integration, `Clock_Slow` is driven by `flag_read` (routed as `CLK_D`).
- `flag_read` is generated by a counter with period `2**Rate_CLK_D`.
- `Rate_CLK_D` is `23` in the current production configuration.

Therefore, with fast clock `f_clk`:

- Effective decimation/update factor: `D = 2^23`
- Effective output update rate: `f_out = f_clk / 2^23`

Important naming clarification:

- `Clock_Slow` is the pulse signal (`std_logic`).
- `23` is the configuration constant (`Rate_CLK_D`), not the value of `Clock_Slow`.

## Behavioral notes

- Decimation cadence is externally controlled by `Clock_Slow`.
- `Qout` changes only on cycles where `Clock_Slow = '1'`.
- Output scaling/truncation behavior is MSB-truncate from the internal 2-stage integrator chain.
- No saturation logic is implemented; arithmetic wraps in two's-complement width.
- `out_rate` and the external `Clock_Slow` cadence should stay consistent.
  If they diverge, internal bit-growth assumptions and effective filter behavior no longer match the intended design point.

## Production-aligned configurations used in rpll

- Frequency slow readout path: `in_bit=12`, `out_bit=56`, `out_rate=23`
- I path slow readout: `in_bit=26`, `out_bit=32`, `out_rate=23`
- Q path slow readout: `in_bit=26`, `out_bit=32`, `out_rate=23`
